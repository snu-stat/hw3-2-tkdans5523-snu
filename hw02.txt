1. MDSR 6장 연습문제 6.6.9

- Replicate the functionality of `make_babynames_dist()` from the mdsr package to wrangle the original tables from the babynames package.

### 답안

- `make_babynames_dist` 함수는 `babynames` 패키지의 `babynames` 데이터프레임과 `lifetables` 데이터프레임을 사용하여 특정 시점(2014년)에 이름별로 태어난 아이들의 수와 생존 확률을 계산하여 새로운 데이터프레임을 생성하는 함수이다.
    - variables of lifetables
        - `x`: 나이 (0세~110세).
        - `lx`: 특정 나이(x)까지 살아남은 인원수 (기준 인원 100,000명당).
        - `ex`: 해당 나이(x)에서의 기대 여명 (앞으로 몇년을 더 살 것으로 기대되는지).
        - `year`: 해당 생존율이 기록된 출생 연도.
    - variables of babynames
        - `year`: 출생 연도.
        - `sex`: 성별 (M 또는 F).
        - `name`: 이름.
        - `n`: 출생 수.

- `?`를 사용하여 함수와 데이터프레임의 구조와 내용을 살펴보면, `make_babynames_dist` 함수는 다음과 같은 과정을 거쳐 데이터를 가공한다. (`?` 커맨드는 생략했다.)
    - `age_today`: 현재(2014년) 기준 나이 ($2014 - $`year`).
    - `alive_prob`: 현재까지 살아있을 확률 (`lx` / 100,000).
    - `life_exp`: 기대 수명 (`x + ex`, 즉 현재 나이 + 남은 수명).
    - `est_alive_today`: 오늘날 살아있을 것으로 추정되는 인원수 (`n *alive_prob`).
    - 'approximate' : lifetables는 10년 단위로 데이터가 존재하기 때문에 `make_babynames_dist()`에서는 기본 함수인 `approx`를 사용한 `approximate` 함수를 만들어서 사용함.

- 이를 참고하여 `make_babynames_dist()`의 기능을 R과 Python으로 각각 구현해보자.

::: {.panel-tabset}

## R

```{r}
# 나이(x)와 출생 연도(year)를 더해 2014년이 되는 데이터만 필터링하여
# year, sex, alive_prob, age_today, life_exp 열을 가지도록 데이터 만들기
lifetables_2014 <- lifetables %>%
  filter(year + x == 2014) %>%
  mutate(
    alive_prob = lx / 100000,    # 10만 명당 생존자 수(lx)를 확률로 변환
    age_today = x,               # 2014년 기준 나이
    life_exp = x + ex            # 기대 수명 (현재 나이 + 남은 수명)
  ) %>%
  select(year, sex, alive_prob, age_today, life_exp)

# 모든 출생 연도에 대해 확률을 채워주기 위해 approx 함수를 활용.
years <- min(lifetables_2014$year):max(lifetables_2014$year) # 1900년부터 2010년까지

# 남성의 연도별 생존 확률 근사 데이터
men <- lifetables_2014 %>%
  filter(sex == "M") %>%
  { approx(x = .$year, y = .$alive_prob, xout = years) } %>% # approx 함수를 사용하여 연도별 생존 확률을 보간
  as_tibble() %>% # approx 함수의 결과는 리스트 형태이므로 tibble로 변환
  rename(year = x, alive_prob = y) %>%
  mutate(sex = "M")

# 여성의 연도별 생존 확률 근사 데이터
women <- lifetables_2014 %>%
  filter(sex == "F") %>%
  { approx(x = .$year, y = .$alive_prob, xout = years) } %>%
  as_tibble() %>%
  rename(year = x, alive_prob = y) %>%
  mutate(sex = "F")

# 남녀 데이터를 행 기준 결합
actuarial_interp <- bind_rows(men, women)

# 데이터 결합 : actuarial_interp가 year, alive_prob, sex 열을 가지고 있고
# babynames 데이터프레임이 year, sex, name, n, prop 열을 가지고 있으므로,
# year과 sex를 기준으로 두 데이터프레임을 결합한다.
MyBabynamesDist <- babynames %>%
  inner_join(actuarial_interp, by = c("year", "sex")) %>%
  # babynames의 각 행에 기록된 출생 수와 actuarial_interp의 alive_prob를 활용하여
  # 연도, 이름, 성별의 조합에 대해 2014년 기준 예상 생존 인원(est_alive_today) 계산
  mutate(
    count_thousands = n / 1000,             # 인원수 (단위: 천 명)
    age_today = 2014 - year,                # 2014년 기준 나이
    est_alive_today = n * alive_prob        # 2014년 기준 예상 생존 인원
  ) %>%
  # 열 순서 맞추기
  select(year, sex, name, n, prop, alive_prob, count_thousands, age_today, est_alive_today)

# 결과 확인
origin_data = make_babynames_dist()
made_data = MyBabynamesDist

# print(summary(origin_data$est_alive_today)) : python의 결과와 비교하기 위한 est_alive_today의 summary
# min : 0, 1st Qu. : 4.966, Median : 8.929, Mean : 150.907, 3rd Qu. : 24.484, Max : 80399.051

all.equal(origin_data, made_data) # TRUE가 나오면 두 데이터프레임이 동일함
```

## Python

```{python}
# Data Load
babynames_df = bn.babynames
lifetables_df = bn.lifetables

# 나이(x)와 출생 연도(year)를 더해 2014년이 되는 데이터만 필터링하여
# year, sex, alive_prob, age_today, life_exp 열을 가지도록 데이터 만들기
lifetables_2014 = (
    lifetables_df
    .filter((pl.col("year") + pl.col("x")) == 2014)
    .with_columns([
        (pl.col("lx") / 100000).alias("alive_prob"),
        pl.col("x").alias("age_today")
    ])
    .select(["year", "sex", "alive_prob", "age_today"])
)

# scipy의 interp1d를 사용하여 R의 approx() 함수와 동일한 기능을 수행.
years_range = np.arange(lifetables_2014["year"].min(), lifetables_2014["year"].max() + 1)

def interpolate_sex(df, sex_label):
    subset = df.filter(pl.col("sex") == sex_label).sort("year")
    # 연도별 생존 확률을 interpolation하는 함수 생성
    f = interp1d(subset["year"], subset["alive_prob"], kind='linear', fill_value="extrapolate")
    return pl.DataFrame({
        "year": years_range.astype(np.int32),
        "alive_prob": f(years_range),
        "sex": sex_label
    })

# 남녀 데이터 보간 후 결합 (R의 bind_rows)
actuarial_interp = pl.concat([
    interpolate_sex(lifetables_2014, "M"),
    interpolate_sex(lifetables_2014, "F")
])

# 데이터 결합 : actuarial_interp가 year, alive_prob, sex 열을 가지고 있고
# babynames 데이터프레임이 year, sex, name, n, prop 열을 가지고 있으므로,
# year과 sex를 기준으로 두 데이터프레임을 결합한다.
MyBabynamesDist = (
    babynames_df
    .with_columns(pl.col("year").cast(pl.Int32)) # 연도를 정수로 변환
    .join(actuarial_interp, on=["year", "sex"], how="inner") # 연도와 성별을 기준으로 결합
    # babynames의 각 행에 기록된 출생 수와 actuarial_interp의 alive_prob를 활용하여
    # 연도, 이름, 성별의 조합에 대해 2014년 기준 예상 생존 인원(est_alive_today) 계산
    .with_columns([
        (pl.col("n") / 1000).alias("count_thousands"),
        # 이름 일치시키기
        (2014 - pl.col("year")).alias("age_today"), 
        (pl.col("n") * pl.col("alive_prob")).alias("est_alive_today")
    ])
    .select([
        "year", "sex", "name", "n", "prop", 
        "alive_prob", "count_thousands", "age_today", "est_alive_today"
    ])
)

# 결과 확인
# R에서의 결과
# min : 0, 1st Qu. : 4.966, Median : 8.929, Mean : 150.907, 3rd Qu. : 24.484, Max : 80399.051
summary_stats = MyBabynamesDist.select("est_alive_today").describe()

result_output = f"""
Summary of Estimated Alive Today (Python):
{summary_stats}
"""
print(result_output)
```

```{python}
import polars as pl

_Teams = pylahman.Teams()
```

```{r}
# pulse_plot 함수 정의
pulse_plot <- function(.data, x_var) {
  p <- ggplot(.data, aes(y = Pulse, x = .data[[x_var]])) +
    geom_jitter(alpha = 0.3) + 
    geom_smooth() +
    labs(
      title = paste("Pulse by", x_var, "in NHANES Dataset"),
      x = x_var,
      caption = "US National Center for Health Statistics (NCHS)"
    ) +
    theme_minimal()

  return(p)
}

# purrr::map을 활용한 Visualization
c("Age", "BMI", "TVHrsDay", "BPSysAve") %>%
  map(pulse_plot, .data = NHANES) %>%
  patchwork::wrap_plots(ncol = 2)
```

## Python

```{python}
# Data Load
NHANES = pl.read_csv('data/NHANES.csv',
    schema_overrides={
        "Age": pl.Float64, 
        "BMI": pl.Float64
    },
    null_values=["NA", " ", "null"],
    infer_schema_length=None
)

#  pulse_plot 함수 정의
def pulse_plot(data, x_var):
    p = (
        ggplot(data, aes(y="Pulse", x=x_var))
        + geom_jitter(alpha=0.3)
        + geom_smooth(method="lowess", color="blue")
        + labs(
            title=f"Pulse by {x_var} in NHANES Dataset",
            x=x_var
        )
        + theme_minimal()
    )
    return p

variables = ["Age", "BMI", "TVHrsDay", "BPSysAve"]
plots = [pulse_plot(NHANES, var) for var in variables]

# plotnine 객체를 patchworklib 객체로 변환
p1, p2, p3, p4 = [pw.load_ggplot(p, figsize=(4, 3)) for p in plots]

# 2x2 그리드로 배치
final_plot = (p1 | p2) / (p3 | p4) 

# 파일 저장 (그림이 계속 깨져서 사진 파일로 저장한 다음 로드 시켰습니다)
# jupyter notebook에서는 제대로 나옴
output_file = "figure/pulse_analysis_plot.png"
final_plot.savefig(output_file, dpi=300) # 그래프 사진 파일로 저장
```

- Use the `group_modify()` function and the Lahman data to replicate one of the baseball records plots (http://tinyurl.com/nytimes-records) from the The New York Times.

### 답안

- NY Times의 plot 중에서 타율 데이터를 `group_modify()`를 활용하여 시각화하는 코드를 구현해보자.
- NY Time에 나온 plot에 년도별 기록된 샘플이 대부분 30개 이하이고 년도의 범위는 1901년부터 2014년까지이고 가장 타율이 낮은 선수는 0.28의 타율을 기록했다.
- 따라서 이를 모사하기 위해 `filter` 함수를 사용하여 x축과 y축 범위에 맞는 데이터만 필터링하였다. ( 완벽히 같지는 않고 대략적인 모사를 코드에 반영하였다. )
- 모든 데이터에 대한 plot을 그리는 것은 시각적으로 복잡해 보이기 때문에 이전 과제를 참고하여 타수 500이상인 선수들에 대해서만 plot을 그렸다.
- 최대한 `group_by`와 `group_modify`만 사용하여 데이터셋을 만들려고 했으나, 여러 시도를 해본 결과 `filter` 함수를 `get_ba` 함수 안에 넣어서 사용하면 시간이 굉장히 오래 걸리기 때문에 부득이하게 `group_modify` 적용 후 `filter` 함수를 사용하였다.

```{r}
get_ba <- function(x) {
  x %>%
    summarize(
      H = sum(H, na.rm = TRUE),
      AB = sum(AB, na.rm = TRUE),
      teamID = paste(unique(teamID), collapse = ", ") # 하나의 연도에 소속팀 2개인 경우
    ) %>%
    mutate(BA = H / AB)
}

# group_modify()로 타율 계산 및 필터링
BA_data <- Batting %>%
  group_by(playerID, yearID) %>%
  group_modify(~ get_ba(.x))
  
# group_modify()가 오래걸려서 코드 수정을 위해 나눠서 작업 수행  
BA_data <- BA_data %>%
  filter(yearID >= 1901) %>% # 1900년 이후 데이터만 사용
  filter(yearID <= 2014) %>% # 2015년 이전 데이터만 사용
  filter(AB >= 500) %>% # 타수 500이상
  filter(BA >= 0.25) %>% # 타율 0.25 이상
  ungroup() # 상위 선수 추출을 위해 ungroup()

# 역대 단일 시즌 타율 상위 3개 추출
top_records <- BA_data %>%

# 텍스트를 달기 위한 데이터: 상위 3개 기록 중 선수별 가장 높은 1개만 남기고 라벨 텍스트 생성

top_labels <- top_records %>%

  slice_max(BA, n = 1, with_ties = FALSE) %>% # 선수별 최고 기록 1개만 추출
```

```{python}
# bmi_plot 정의 : 수업 참고
def bmi_plot(data, x_var):
    p = (
        ggplot(data, aes(y="BMI", x=x_var)) 
        + geom_jitter(alpha=0.3)
        + geom_smooth(method="lowess", color="blue")      
        + labs(
            title=f"BMI by {x_var} in NHANES",
            caption="US National Center for Health Statistics (NCHS)"
        )
    )
    return p

# 그래프 nrow x ncol 형태로 묶어주는 함수
def wrap_plots(plot_list, nrow, ncol, figsize=(3, 3)):
    pw_plots = [pw.load_ggplot(p, figsize=figsize) for p in plot_list]
    
    rows = []
    idx = 0
    # 행(row) 단위로 먼저 그래프를 가로(|)로 병합
    for r in range(nrow):
        if idx >= len(pw_plots): break
        current_row = pw_plots[idx]
        idx += 1
        for c in range(1, ncol):
            if idx >= len(pw_plots): break
            current_row = current_row | pw_plots[idx]
            idx += 1
        rows.append(current_row)
        
    # 완성된 행들을 세로(/)로 병합
    grid = rows[0]
    for r in rows[1:]:
        grid = grid / r
        
    return grid

# Data Load
NHANES = pl.read_csv('data/NHANES.csv',
    null_values=["NA", " ", "null", ""], 
    infer_schema_length=None
)

#  모든 변수(76개)와 BMI 간의 산점도 그리기 (19 x 4 배열)
all_vars = NHANES.columns
plot_all_list = [bmi_plot(NHANES, var) for var in all_vars]

# 19 x 4로 배치
grid_all = wrap_plots(plot_all_list, nrow=19, ncol=4, figsize=(3.75, 2.6))
grid_all.savefig("figure/3P_plot_all.png", dpi=300)
```

- Problem 8 (Medium): A data analyst received permission to post a data set that was scraped from a social media site. The full data set included name, screen name, email address, geographic location, IP (internet protocol) address, demographic profiles, and preferences for relationships. Why might it be problematic to post a deidentified form of this data set where name and email address were removed?

::: {.callout-note appearance="minimal"}
## Answer
이름과 이메일을 제거하는 것만으로는 충분하지 않다. IP 주소만으로 사용자의 대략적인 거주지나 직장 위치가 노출 될 수 있으며 사실상 식별가능한 정보이다. 거기에 위치 정보, 인구통계적 특성 등이 결합되면 외부 데이터(다른 SNS의 공개 프로필 등)와 대조하여 특정 개인을 식별해낼 수 있기 때문에 개인 정보 노출의 위험이 있다. 특히 관계 선호도와 같은 민감한 정보가 포함되어 있어, 재식별 시 당사자에게 심각한 개인정보 피해를 줄 위험이 크다. 따라서 이러한 데이터 셋을 게시하는 것은 매우 위험한 행위가 될 수 있다.

2. MDSR 8장 연습문제 8.12.10

- Problem 10 (Hard): An investigative team wants to winnow the set of variables to include in their final multiple regression model. They have 100 variables and one outcome measured for n = 250 observations.
- They use the following procedure:
    1. Fit each of the 100 bivariate models for the outcome as a function of a single predictor, then
    2. Include all of the significant predictors in the overall model.
- What does the distribution of the p-value for the overall test look like, assuming that there are no associations between any of the predictors and the outcome (all are assumed to be multivariate normal and independent). Carry out a simulation to check your answer.

### 답안

- 문제에서 요구하는 시뮬레이션 과정은 다음과 같다.
    1. 101개의 변수(`Y` 포함)와 250개의 관측치를 가지는 데이터셋을 생성한다. (문제에서 특정 분포에 대한 요구사항이 없기 때문에 정규분포에서 무작위로 생성한다.)
    2. 각 변수에 대하여 `Y`에 대한 단일 회귀분석을 수행하여 p-value를 추출한다. 
    3. 유의수준 0.05 미만인 변수들만 선택하여 최종 다중 회귀분석을 수행한다. (문제에서 p-value에 대한 유의수준이 설정되어 있지 않지만, 일반적으로 0.05를 사용한다.)
    4. 최종 모델의 전체 모형 유의성 검정(F-test)에서 p-value를 추출하여 저장한다.
- 이 과정을 100번 반복하여 최종적으로 전체 모형의 p-value 분포를 시각화한다.

::: {.panel-tabset}

## R

```{r}
# 시뮬레이션 세팅
set.seed(123) # 시드 고정
n_sim <- 100
n_obs <- 250        
n_vars <- 100       
overall_pvals <- numeric(n_sim) 

for (i in 1:n_sim) {
  # 정규분포에서 랜덤 샘플링
  Y <- rnorm(n_obs)
  X <- matrix(rnorm(n_obs * n_vars), nrow = n_obs, ncol = n_vars)
  
  # 개별 변수들에 대해 회귀분석 수행 후 p-value 추출
  bivariate_pvals <- numeric(n_vars)
  for (j in 1:n_vars) {
    fit <- lm(Y ~ X[, j])
    bivariate_pvals[j] <- summary(fit)$coefficients[2, 4] 
  }
  
  # 유의수준 0.05 미만인 변수들만 선택
  selected_vars <- which(bivariate_pvals < 0.05)
  
  # 선택된 변수들로 다중 회귀분석 수행
  if (length(selected_vars) > 0) {
    X_selected <- X[, selected_vars, drop = FALSE]
    final_fit <- lm(Y ~ X_selected)
    
    # 전체 모형의 유의성 검정(F-test) p-value 계산
    f_stat <- summary(final_fit)$fstatistic
    overall_pvals[i] <- pf(f_stat[1], f_stat[2], f_stat[3], lower.tail = FALSE)
  } else {
    # 우연히 유의미한 변수가 하나도 나오지 않았을 경우 결측치 처리
    overall_pvals[i] <- NA 
  }
}

# 결측치 제거
overall_pvals <- na.omit(overall_pvals)

summary(overall_pvals) # p-value 분포 요약 통계량 확인
```

## Python

```{python}
# 시뮬레이션 세팅
np.random.seed(123) # 시드 고정
n_sim = 100
n_obs = 250        
n_vars = 100       
overall_pvals = [] 

for i in range(n_sim):
    Y = np.random.randn(n_obs)
    X = np.random.randn(n_obs, n_vars)
    
    # 개별 변수들에 대해 회귀분석 수행 후 p-value 추출
    bivariate_pvals = np.zeros(n_vars)
    for j in range(n_vars):
        X_single_with_const = sm.add_constant(X[:, j])
        fit_single = sm.OLS(Y, X_single_with_const).fit()
        bivariate_pvals[j] = fit_single.pvalues[1]
        
    # 유의수준 0.05 미만인 변수들만 선택
    selected_vars = np.where(bivariate_pvals < 0.05)[0]
    
    # 선택된 변수들로 다중 회귀분석 수행
    if len(selected_vars) > 0:
        X_selected = X[:, selected_vars]
        
        X_selected_with_const = sm.add_constant(X_selected)
        final_fit = sm.OLS(Y, X_selected_with_const).fit()
        
        # 전체 모형의 유의성 검정(F-test) p-value 계산 및 저장
        overall_pvals.append(final_fit.f_pvalue)
    else:
        # 우연히 유의미한 변수가 하나도 나오지 않았을 경우 결측치 처리
        overall_pvals.append(np.nan)

# 결측치 제거
overall_pvals = np.array(overall_pvals)
overall_pvals = overall_pvals[~np.isnan(overall_pvals)]

# p-value 분포 요약 통계량 확인
print(pd.Series(overall_pvals).describe())
```
:::

- 요약 통계량 결과를 바탕으로 도출할 수 있는 결론은 다음과 같이 정리할 수 있다.
    - 모든 독립변수가 결과 변수와 실제 관련이 없으므로, 정상적인 상황이라면 p-value는 0과 1 사이에서 균등 분포(Uniform Distribution)를 보여야 한다.
    - 그러나, 시뮬레이션 결과를 보면 최종 다중 회귀모형의 전체 유의성 검정 p-value의 최댓값(Max)조차 약 0.05 미만으로 나타나며 모든 결과가 유의수준 0.05 미만으로 극단적으로 작게 나타난 것을 확인할 수 있다.
    - 이는 종속변수와 실제 연관성이 전혀 없는 데이터로 모델을 구성했음에도 불구하고, 최종 모델이 통계적으로 매우 유의미하다는 제1종 오류가 도출됨을 보여준다.
    - 100개의 무의미한 변수를 개별적으로 검정하면 순전히 우연에 의해 약 5개(5%)의 변수가 유의미한 것으로 판별되는데, 이렇게 우연히 유의성을 보인 변수들만 골라 최종 다중 회귀를 돌리면 전체 모델의 유의성(F-통계량)이 인위적으로 부풀려지게 된다.
    - 결론적으로, 사전에 개별 p-value만을 기준으로 변수를 선택하여 최종 모델을 구축하는 이러한 방식은 심각한 통계적 왜곡을 초래하여 무의미한 모델조차 완벽한 모델처럼 보이게 만들므로 지양해야 할 분석 절차임을 시사한다.

- The `NHANES` data set in the `NHANES` package includes survey data collected by the U.S. National Center for Health Statistics (NCHS), which has conducted a series of health and nutrition surveys since the early 1960s.
    1. An investigator is interested in fitting a model to predict the probability that a female subject will have a diagnosis of diabetes. Predictors for this model include age and BMI. Imagine that only 1/10 of the data are available but that these data are sampled randomly from the full set of observations (this mechanism is called “Missing Completely at Random,” or MCAR). What implications will this sampling have on the results?
    2. Imagine that only 1/10 of the data are available but that these data are sampled from the full set of observations such that missingness depends on age, with older subjects less likely to be observed than younger subjects (this mechanism is called “Covariate Dependent Missingness,” or CDM). What implications will this sampling have on the results?
    3. Imagine that only 1/10 of the data are available but that these data are sampled from the full set of observations such that missingness depends on diabetes status (this mechanism is called “Non-Ignorable Non-Response,” or NINR). What implications will this sampling have on the results?

```{python}
X_train_ninr = sm.add_constant(nhanes_ninr.select(["Age", "BMI"]).to_numpy())

# 로지스틱 회귀분석
ninr_model = sm.Logit(Y_train_ninr, X_train_ninr).fit(disp=False)

# Train Accuracy
train_acc_ninr = np.mean((ninr_model.predict(X_train_ninr) > 0.5).astype(int) == Y_train_ninr)

# Test Accuracy
Y_test_ninr = nhanes_test_ninr.select("Diabetes_binary").to_numpy().ravel()
X_test_ninr = sm.add_constant(nhanes_test_ninr.select(["Age", "BMI"]).to_numpy())
test_acc_ninr = np.mean((ninr_model.predict(X_test_ninr) > 0.5).astype(int) == Y_test_ninr)

# 결과 출력 : summary 및 accuracy
target_counts_ninr = nhanes_ninr.group_by("Diabetes_binary").len().sort("Diabetes_binary")

result_ninr = f"""Sample counts by Diabetes status:
{target_counts_ninr}

Summary of Regression (NINR - 8:2 Ratio)
{ninr_model.summary(xname=['Intercept', 'Age', 'BMI']).tables[1]}

Train accuracy : {train_acc_ninr * 100:.2f}%
Test  accuracy: {test_acc_ninr * 100:.2f}%"""

print(result_ninr)
```

- 해당 샘플링 결과에서도 다른 상황에 비해서 계수 추정치의 차이가 보이고, p-value가 높아지는 경향이 나타난다. 이는 당뇨 여부에 따라 샘플링이 달라지는 상황에서, 당뇨병과 나이 또는 BMI 사이의 실제 연관성이 존재하더라도 샘플링 편향으로 인해 통계적 검정력이 감소하기 때문으로 해석할 수 있다.

- 적합 뒤에는 실제로 쓰인 contrast 행렬을 확인하고, `summary`로 계수별 t검정, `confint`로 95% 신뢰구간, `anova`로 기간 요인 전체의 유의성을 살펴보자.

## R

```{r}
# 모델 적합
fit_ba <- lm(BA ~ period, data = batting_df)

# 직교다항 코딩행렬 확인
BA_contrast <- contrasts(batting_df$period)

# Summary
BA_summary <- summary(fit_ba)

# 신뢰구간 계산
BA_ci <- confint(fit_ba, level = 0.95)

# ANOVA
BA_anova <- anova(fit_ba)

# 결과 출력
result_ba <- paste0(
  "Contrast matrix (period, contr.poly):\n",
  paste(capture.output(BA_contrast), collapse = "\n"), "\n\n",
  "Summary of lm(BA ~ period):\n",
  paste(capture.output(BA_summary), collapse = "\n"), "\n\n",
  "95% CI for coefficients:\n",
  paste(capture.output(BA_ci), collapse = "\n"), "\n\n",
  "F-statistics = ", BA_anova$F[1], "\n"
```

## Python

```{python}
import pandas as pd
import statsmodels.formula.api as smf
from statsmodels.stats.anova import anova_lm
from patsy import dmatrix

# 앞 청크에서 만든 Polars `batting_df`를 그대로 쓰고, 선형모형에 넣을 열만 고른다.
ba_for_model = batting_df.select(["BA", "period"])

# statsmodels·patsy는 Polars를를 직접 받지 않으므로, pandas로 변환 후 진행
pd_ba = ba_for_model.to_pandas()
pd_ba["period"] = pd.Categorical(pd_ba["period"], categories=period_levels, ordered=True)

# 모델 적합
fit_ba = smf.ols("BA ~ C(period, Poly)", data=pd_ba).fit()

# 직교다항 코딩행렬 확인 (기간 수준별 다항 대비값; R의 contrasts(period)에 대응)
Z_poly = dmatrix("0 + C(period, Poly)", pd_ba, return_type="dataframe")
BA_contrast = (
    pd.concat([pd_ba[["period"]].reset_index(drop=True), Z_poly.reset_index(drop=True)], axis=1)
    .drop_duplicates(subset=["period"])
    .sort_values("period")
    .to_string()
)

# Summary
BA_summary = fit_ba.summary().as_text()

# 신뢰구간 계산
BA_ci = fit_ba.conf_int(alpha=0.05)

# ANOVA
BA_anova = anova_lm(fit_ba, typ=2)
BA_F_stat = BA_anova["F"].dropna().iloc[0]

# 결과 출력
result_ba = (
    "Contrast matrix (period, contr.poly):\n"
    + BA_contrast
    + "\n\nSummary of OLS(BA ~ C(period, Poly)):\n"
    + BA_summary
    + "\n\n95% CI for coefficients:\n"
    + BA_ci.to_string()
    + "\n\nF-statistics = "
    + str(BA_F_stat)
    + "\n"
)
print(result_ba)
```
:::

- 직교다항 코딩행렬은 세 기간을 등간격 순서로 놓았을 때의 선형(`.L`)·이차(`.Q`) 대비 가중치를 보여 준다.
- `period.L`의 계수는 음수이고 p값이 매우 작아, 기간 순서를 따라 잡은 선형 성분만으로도 평균 타율에 차이가 있음을 시사한다. 95% 신뢰구간 [-0.00374, -0.00132]이 0을 포함하지 않으므로, 정규오차 가정 하에서 선형 다항항은 통계적으로 유의하다.
- `period.Q`의 계수도 음수이며 p값이 2×10⁻¹⁶ 미만으로 더 강하게 유의하고, 신뢰구간도 0을 벗어난다. 이는 세 시기 평균 타율의 차이가 선형적인 패턴보다는 이차 곡선의 형태로 나타난다는 뜻으로 해석할 수 있다(구체적 방향은 그룹별 표본 평균과 함께 보는 것이 좋다).
- 분산분석에서 기간 요인에 대한 F=76.27 (df=2), p < 2.2×10⁻¹⁶이므로, 두 다항항을 묶어 볼 때 세 기간의 평균 타율이 동일하다는 귀무가설은 기각된다. 다만 $R^2 \approx 0.006$으로, 기간만으로는 선수–연도별 타율 변동의 극히 일부만 설명되며 다른 통제 요인이 있을 것으로 예상된다.
- a항에서 시각화한 각 시기별 분포를 고려할 때, b항에서 전제한 오차항의 정규분포 가정이 엄밀한 의미에서 완벽히 타당하다고 보기는 어렵다. 타율은 본질적으로 0과 1 사이로 제한된 비율 데이터이므로, 특정 구간(예: 0.200~0.300)에 데이터가 크게 밀집되어 완벽한 좌우 대칭의 종 모양을 띠기보다는 비대칭성(왜도)을 보이거나 양 극단에 이상치가 존재할 가능성이 높기 때문이다.

---

- 이제 bootstrap 방법으로 b항과 같은 분석을 해보자. (타율 평균 차이에 대한 신뢰구간을 계산)
- 반복 횟수는 1000번으로 두고, 매 반복마다 관측을 복원 추출한 표본에 모형을 다시 적합하여 절편·`period.L`·`period.Q` 계수를 저장한다.
- 저장된 계수의 분포에서 **백분위수 2.5%, 97.5%**를 취해 각 계수의 95% 신뢰구간을 만들고, b항에서의 신뢰구간과 비교한다.

::: {.panel-tabset}

## R

```{r}
# Baseline model
fit_ref <- lm(BA ~ period, data = batting_df)
nm <- names(coef(fit_ref))
p <- length(nm)
ci_norm <- confint(fit_ref, level = 0.95)

# 시뮬레이션 세팅
set.seed(123)
n_sim <- 1000
n_obs <- nrow(batting_df)
boot_coef <- matrix(NA_real_, nrow = n_sim, ncol = p, dimnames = list(NULL, nm))

for (i in seq_len(n_sim)) {
  boot_idx <- sample.int(n_obs, n_obs, replace = TRUE)
  boot_df <- batting_df[boot_idx, ]
  fit_i <- lm(BA ~ period, data = boot_df)
  boot_coef[i, ] <- coef(fit_i)
}

# 부트스트랩 95% 신뢰구간
ci_boot <- t(apply(boot_coef, 2, quantile, probs = c(0.025, 0.975)))
colnames(ci_boot) <- c("2.5%", "97.5%")

# 결과 출력
out_boot <- paste(capture.output(print(ci_boot)), collapse = "\n")
out_norm <- paste(capture.output(print(ci_norm)), collapse = "\n")

result_boot <- paste0(
  "Bootstrap 95% CI (coefficients, percentile):\n",
  out_boot, "\n\n",
  "Baseline model 95% CI:\n",
  out_norm, "\n"
)
cat(result_boot)
```

## Python

```{python}
import numpy as np
import pandas as pd
import statsmodels.formula.api as smf

# Baseline model
fit_ref = smf.ols("BA ~ period", data=pd_ba).fit()
nm = fit_ref.params.index
p = len(nm)

ci_norm = fit_ref.conf_int(alpha=0.05)
ci_norm.columns = ["2.5%", "97.5%"]


# 시뮬레이션 세팅
np.random.seed(123)
n_sim = 100
n_obs = len(pd_ba) # 24880

boot_coef = np.zeros((n_sim, p))

for i in range(n_sim):
    boot_df = pd_ba.sample(n=n_obs, replace=True)
    fit_i = smf.ols("BA ~ period", data=boot_df).fit()
    boot_coef[i, :] = fit_i.params.values

boot_coef_df = pd.DataFrame(boot_coef, columns=nm)

# 부트스트랩 95% 신뢰구간 (Percentile method)
ci_boot = boot_coef_df.quantile([0.025, 0.975]).T
ci_boot.columns = ["2.5%", "97.5%"]

# 결과 출력
result_boot = (
    "Bootstrap 95% CI (coefficients, percentile):\n"
    + ci_boot.to_string()
    + "\n\nBaseline model 95% CI:\n"
    + ci_norm.to_string()
    + "\n"
)
print(result_boot)
```

:::

- 부트스트랩을 이용한 구간 추정치를 b항의 결과와 비교했을 때 매우 유사한 결과가 나온 것을 확인할 수 있다.
- 이러한 결과가 나타난 핵심적인 이유는 분석에 사용된 표본의 크기(n=24880)가 크기 때문에 우리가 익히 알고 있는 중심극한정리(Central Limit Theorem)가 적용된 사례라고 생각된다.
- 모집단 타율 데이터의 분포가 완벽한 정규분포를 따르지 않고 비대칭성을 띠더라도, 표본의 크기($n$)가 충분히 크면 표본평균의 분포는 근사적으로 정규분포를 띠게 된다.
- 따라서 정규분포를 엄격하게 가정하고 이론적 수식을 통해 구한 b항의 신뢰구간과, 실제 데이터를 바탕으로 무작위 복원 추출을 반복해 경험적으로 구한 부트스트랩 신뢰구간 모두 결과적으로는 정규분포의 형태에 수렴하게 된다.
- 결론적으로 두 방식 모두 충분히 큰 표본 하에서 중심극한정리의 영향을 받아 동일한 확률적 특성을 공유하게 되므로, 최종적으로 추정된 신뢰구간이 거의 일치하는 것으로 보인다.

```{python}
# Data Load
batting = pl.from_pandas(pylahman.Batting())

# 순서 정의
period_levels = ["1_Pre", "2_Steroid", "3_Post"]

# Polars Data Wrangling
hr_top1_pl = (
    batting
    .filter(pl.col("yearID") >= 1982)
    .drop_nulls("HR") # 결측치 제거
    .group_by(["playerID", "yearID"])
    .agg([
        pl.col("HR").sum()
    ])
    .with_columns(
        period=pl.when(pl.col("yearID") <= 1993).then(pl.lit("1_Pre"))
              .when(pl.col("yearID") <= 2005).then(pl.lit("2_Steroid"))
              .otherwise(pl.lit("3_Post"))
    )
    .with_columns(
        pl.col("period").cast(pl.Enum(period_levels))
    )
    .filter(
        pl.col("HR") >= pl.col("HR").quantile(0.99).over("period")
    )
)

hr_top1_df = hr_top1_pl.to_pandas() # statsmodels는 Polars를 직접 받지 않으므로, pandas로 변환 후 진행
hr_top1_df['period'] = pd.Categorical(hr_top1_df['period'], categories=period_levels, ordered=True)
hr_top1_df['HR'] = pd.to_numeric(hr_top1_df['HR'])
hr_top1_df['yearID'] = pd.to_numeric(hr_top1_df['yearID'])

# ==========================================

print(p_density)

fit_hr = smf.ols("HR ~ C(period, Poly)", data=hr_top1_df).fit()
```

### 2-2 : (a)항

- 시각화 결과를 보면, 각 시기별 상위 1% 홈런수(HR) 분포는 정규분포를 따른다고 보기 어렵다고 생각한다.
- 홈런수 상위 1%에 해당하는 극단적인 사례들만 추출하였기 때문에, 정규분포의 종 모양보다는 특정 구간에 치우치거나 오른쪽으로 꼬리가 길게 늘어지는 비대칭적인 형태를 보이는 것은 자연스러운 결과라고 생각한다.

### 2-2 : (b)항

- `period.L`의 계수는 양수(약 2.37)이고 p값이 매우 작아 통계적으로 유의하다. 95% 신뢰구간 [1.57, 3.17] 역시 0을 포함하지 않는다. 이는 기간이 지남에 따라 전반적으로 홈런수가 증가하는 선형적 추세가 일부 존재함을 의미한다.
- `period.Q`의 계수는 음수(약 -5.86)이며 p값이 2×10⁻¹⁶ 미만으로 매우 강하게 유의하고, 신뢰구간 [-6.73, -5.00]도 0을 크게 벗어난다.
- 문제 2-1은 타율에 대한 데이터로 데이터의 수치 자체가 0에서 1로 제한되어 있어서 회귀 계수의 값이 직관적인 차이가 있다고 보긴 어려웠지만 홈런 수에 대한 데이터로는 계수만으로 직관적인 차이가 있을 것으로 예상할 수 있다.
- 분산분석에서 기간 요인에 대한 F=93.68 (df=2), p < 2.2×10⁻¹⁶이므로, 두 다항항을 묶어 볼 때 세 기간의 평균 홈런수가 동일하다는 귀무가설은 기각된다.
- a항에서 시각화한 분포를 고려할 때, 극단값들의 모음인 상위 1% 홈런수 데이터는 뚜렷한 우측 꼬리 비대칭성을 띠므로 b항에서 전제한 오차항의 정규분포 가정이 엄밀한 의미에서 완벽히 타당하다고 보기는 어렵다고 생각한다.

### 2-2 : (c)항

- 부트스트랩을 이용한 구간 추정치를 b항의 결과와 비교했을 때 매우 유사한 결과가 나온 것을 확인할 수 있다.
- 이또한 문제 2-1과 마찬가지로 중심극한정리(Central Limit Theorem)가 적용된 사례라고 생각된다.
- 상위 1%만 추려내어 표본의 크기($n = 571$)가 이전 경우에 비해 크게 줄어들었음에도 불구하고, 여전히 충분히 큰 표본이기 때문에 두 구간 추정치가 유사하게 나온 것이라고 판단된다.

## 문제 2-3

문제 2-2에서 추출한 관측치를 사용하여 다음 질문에 답하라.

1982년부터 2025년까지 홈런수의 분포가 어떻게 변해왔는지 시각화하고 홈런수를 반응변수로, 연도(`yearID`)를 독립변수로 하는 회귀분석을 실시하라. 연도의 효과가 거의 유의하지 않게 나온다면 분석을 올바르게 한 것이다. 타격에 대한 유력한 이론은 시간이 지날수록 홈런의 개수가 증가해 왔다는 것인데, 이 분석이 그러한 이론을 반증한다고 할 수 있는가? 아니라고 생각한다면 적절한 교락(confounding)요인을 회귀모형에 포함시켜 다시 분석을 한 뒤에 결과가 어떻게 변하는지 비교하라.

### 답안

- 먼저 연도별 상위 1% 홈런수 산점도 및 선형 추세선 시각화를 해보자.


::: {.panel-tabset}

## R

```{r}
# 연도별 상위 1% 홈런수 산점도 및 선형 추세선 시각화
p_trend <- ggplot(hr_top1_df, aes(x = yearID, y = HR)) +
  geom_point(alpha = 0.5, position = position_jitter(width = 0.2)) +
  geom_smooth(method = "lm", color = "red", se = TRUE) +
  labs(
    title = "Trend of Top 1% Home Runs (1982-2025)",
    x = "Year",
    y = "Home Runs"
  ) +
  theme_minimal() +
  theme(plot.title = element_text(hjust = 0.5, face = "bold"))

p_trend

# 선형 회귀분석
fit_year <- lm(HR ~ yearID, data = hr_top1_df)

print(summary(fit_year))
```

## Python

```{python}
# 연도별 상위 1% 홈런수 산점도 및 선형 추세선
p_trend = (
    ggplot(hr_top1_df, aes(x='yearID', y='HR')) +
    geom_jitter(alpha=0.5, width=0.2) + 
    geom_smooth(method="lm", color="red", se=True) +

        title="Trend of Top 1% Home Runs (1982-2025)",

p_trend

fit_year = smf.ols("HR ~ yearID", data=hr_top1_df).fit()

print(fit_year.summary().as_text())
```

:::

### 시각화 및 회귀 분석 결과
- 시각화 결과를 보면, 1982년부터 2025년까지 상위 1% 타자들의 홈런수는 연도에 따라 뚜렷한 선형적 증가 추세를 보인다고 하기엔 어려워 보인다.
- 회귀분석 결과를 보아도 p-value가 0.05 미만이므로 연도(`yearID`)의 효과가 통계적으로 유의하긴 하지만, 결정계수($R^2$)가 0.009에 불과하여 모델의 설명력이 매우 낮다.
- 즉, 표본의 크기(n=571)가 커서 실질적 의미가 없는 미세한 기울기마저 유의한 것으로 계산된 것이라고 판단된다.

### 스테로이드 시대와의 연관성 추측
- 문제 2-1와 2-2에서 연도를 세 시기로 나누어 분석했을 때, 2차 항이 매우 유의하게 나타났던 것을 확인했었다.
- 그리고 이 3개의 시기는 연도를 기준으로 분류한 것이기 때문에, 연도(`yearID`)와 시기(`period`)는 구조적으로 연관된 변수(다중공선성)라고 볼 수 있다.
- 따라서 시기(`period`) 변수를 추가하여 다중 회귀분석을 실시해보자.

::: {.panel-tabset}

## R

```{r}
fit_year_period <- lm(HR ~ yearID + period, data = hr_top1_df)

print(summary(fit_year_period))
```

## Python

```{python}
fit_year_period = smf.ols("HR ~ yearID + C(period, Poly)", data=hr_top1_df).fit()

print(fit_year_period.summary().as_text())
```
:::

- 시기 변수를 포함한 분석 결과, 모형의 설명력($R^2$)이 0.009에서 0.251로 상승하였으며, 스테로이드 시대의 홈런 폭증을 나타내는 이차항(`period.Q`)이 강한 음수(-5.98)를 띠며 통계적으로 매우 유의하게 나타났다.
- 연도(`yearID`)와 `period.L`의 경우, 두 변수 간의 구조적 연관성(다중공선성)으로 인해 p-value 자체가 상승(0.116)하여 유의하지 않는 변수로 볼 수 있다.
- 결국 시기에 따른 제곱차항이 매우 유의하여 전체 모델의 설명력이 크게 증가한 반면, 연도 자체의 효과는 유의하지 않게 나오는 결과가 나타났다.
- 결론적으로 교락 요인을 누락한 단순 회귀분석 결과는 매년 홈런의 개수가 증가한다는 타격 이론을 반증하지 못한다.

# 3부  데이터 분석 기술

숙제 1에서는 공식 GitHub 학생 개발자 팩에 가입하고 GitHub Pro 및 GitHub Codespaces 사용을 신청했다. 여기서는 숙제 2 제출용 GitHub 저장소에 작업한 Quarto markdown 소스 파일(`hw02.qmd`)을 올리면 GitHub에서 자동으로 HTML 파일 및 주피터 노트북 파일(`.ipynb`)을 만들고 이것을 [GitHub Pages](https://docs.github.com/en/pages/quickstart)에서 웹페이지로 보이도록 설정하는 것을 연습한다.

## 문제 3-1. Quarto 프로젝트 파일 설정

로컬 저장소 최상위 디렉토리에 `_quarto.yml` 파일에 다자음 내용을 추가한다. 파일이 없다면 새로 만들어 추가한다.
```{yml}
project:
  type: website
  output-dir: _site

format:
  html:
    embed-resources: true
  ipynb: default  # HTML 빌드 시 ipynb 파일도 함께 생성하도록 설정
```

`hw02.qmd`를 로컬 저장소에서 컴파일하면 `_site` 디렉토리가 생성되고 그 안에 HTML 및 수반되는 파일들이 만들어지는지 확인하라.

## 문제 3-2. GitHub Actions 워크플로우 생성

GitHub 저장소에 로컬 저장소가 push될 때마다 `.qmd` 파일 컴파일 등의 작업을 자동으로 하게 하는 일은 GitHub Actions에서 담당한다. 로컬 저장소의 `.github/workflows/` 디렉토리(없으면 만든다) 안에 `publish.yml` 파일을 아래와 같이 만든다.
```{yml}
name: Quarto Publish (R & Custom Packages)
on:
  push:
    branches: [ main ]

jobs:
  build-deploy:
    runs-on: ubuntu-latest
    permissions:
      contents: write
    steps:
      - name: Check out repository
        uses: actions/checkout@v4

      - name: Install System Dependencies
        run: |
          sudo apt-get update

      - name: Setup Miniconda
        uses: conda-incubator/setup-miniconda@v3
        with:
          auto-update-conda: true
          python-version: '3.10'
          activate-environment: r-reticulate
          channels: conda-forge,defaults

      - name: Install Python dependencies via Conda
        run: |
          conda install -n r-reticulate numpy pandas matplotlib polars
        # 추가로 필요한 패키지 설치
        shell: bash -l {0}

      - name: Set up R
        uses: r-lib/actions/setup-r@v2
        with:
          use-public-rspm: true 

      - name: Set up Quarto
        uses: quarto-dev/quarto-actions/setup@v2

      - name: Install R Packages
        env:
          GITHUB_PAT: ${{ secrets.GITHUB_TOKEN }}
        run: |
          install.packages(c("reticulate", "knitr", "rmarkdown"))
        # 추가로 필요한 패키지 설치
        shell: Rscript {0}

      - name: Render Quarto
        run: |
          export RETICULATE_PYTHON=$(conda info --base)/envs/r-reticulate/bin/python
          quarto render
        shell: bash -l {0}

      - name: Deploy to GitHub Pages
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./_site
          publish_branch: gh-pages
```

추가로 `hw02.qmd` 파일을 컴파일하기 위해 필요한 R과 Python 패키지들을 "# 추가로 필요한 패키지 설치" 부분에 기술한다.

## 문제 3-3. GitHub Pages 설정 변경

저장소를 웹페이지로 활용하는 부분은 GitHub Pages가 담당한다. 

1. 문제 3-2까지의 작업을 push한 후, GitHub (원격)저장소에 `gh-pages` 브랜치가 생성됐는지 확인하라.

2. GitHub 저장소의 Settings > Pages 메뉴로 이동하여 Build and deployment > Branch 섹션에서 브랜치를 `gh-pages`로 선택하고 폴더를 `/ (root)`로 설정한 뒤 저장하라.

3. GitHub 저장소의 Actions 메뉴로 이동하여 2개의 워크플로우가 무사히 종료되었는지 확인하라.

4. 다시 GitHub 저장소의 Settings > Pages 메뉴로 이동하여 상단에 "Your site is live at https://..." 안내가 보이는지 확인하라. 이 곳의 URL이 hw02.qmd로 생성한 웹페이지의 주소다. "Visit site" 버튼을 클릭하여 웹페이지를 브라우저에서 볼 수 있는지 확인하라.

## 문제 3-4. 반복

Quarto 프로젝트 설정에서 주피터 노트북 파일도 함께 생성하도록 설정하였다. 웹페이지에서 노트북을 내려받을 수 있도록 하기 위해 첫 페이지를 변경하고자 한다. 
이를 위해 `README.md` 파일을 로컬 저장소 최상위 디렉토리에 다음과 같이 만들자.

```{markdown}
# 숙제 2

이름: [아무개]
학번: [나의 학번]

이 숙제의 상세 분석 결과는 아래 링크에서 확인하실 수 있습니다.

* [분석 리포트 (HTML)](./hw02.html) 
* [주피터 노트북 (ipynb)](./hw02.ipynb)
```

이제 Quarto가 README.md를 첫 페이지로 인식하도록 하기 위해 `_quarto.yml`의 "Render Quarto" 앞에 다음을 추가하자.
```{yml}
      - name: Prepare README as Index
        run: cp README.md index.md
```

작업을 GitHub 원격 저장소로 push한 후 문제 3-3의 3, 4번 과정을 반복하라.

## 3번 문제 답안

- 문제 3-1부터 3-4까지의 작업을 모두 마치려고 하였으나, GitHub Actions 워크플로우에서 "Render Quarto" 단계가 계속해서 실패하는 문제가 발생했습니다.
- 문제를 해결하기 위해 여러 가지 시도를 해보았으나, 과제의 원본 파일로 커밋을 진행해도 계속해서 같은 단계에서 실패했습니다. 
- 그래도 `_quarto.yml`과 `publish.yml` 및 `README.md` 파일은 과제에서 제시된 내용에 맞게 작성하여 커밋하였고, GitHub Pages 설정도 완료하였습니다.
- 여러번 시도했다는 것을 증빙하기 위해 초기 파일로 커밋한 결과물 제출하겠습니다.
